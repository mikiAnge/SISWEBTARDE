Guía de Git

1. Configuración Inicial

git config --global user.name "Tu Nombre"  # Configura tu nombre de usuario en Git
git config --global user.email "tuemail@example.com"  # Configura tu correo electrónico

2. Crear un Repositorio

git init  # Crea un nuevo repositorio de Git en el directorio actual
git clone [URL]  # Clona un repositorio existente desde una URL

3. Trabajar con Archivos

git add [archivo]  # Añade un archivo al área de preparación (staging area)
git add .  # Añade todos los archivos modificados al área de preparación
git commit -m "Mensaje del commit"  # Crea un commit con los cambios añadidos y un mensaje descriptivo
git status  # Muestra el estado de los archivos en el directorio de trabajo y el área de preparación
git diff  # Muestra las diferencias entre los archivos modificados y su versión anterior

4. Ramas (Branches)

git branch  # Lista todas las ramas en el repositorio
git branch [rama]  # Crea una nueva rama
git checkout [rama]  # Cambia a la rama especificada
git merge [rama]  # Fusiona la rama especificada con la rama actual

5. Trabajar con Repositorios Remotos

git remote add origin [URL]  # Asocia un repositorio remoto con el nombre "origin"
git push origin [rama]  # Sube los commits de la rama local especificada al repositorio remoto
git fetch  # Descarga cambios desde el repositorio remoto sin aplicarlos
git pull  # Descarga y aplica los cambios desde el repositorio remoto

6. Deshacer Cambios

git reset [archivo]  # Quita el archivo del área de preparación
git reset --soft [commit]  # Mueve el puntero del commit actual a uno anterior, manteniendo los cambios en el área de preparación
git reset --hard [commit]  # Mueve el puntero del commit actual a uno anterior, eliminando todos los cambios realizados desde ese commit

7. Historial y Navegación

git log  # Muestra el historial de commits
git show [commit]  # Muestra los cambios introducidos en un commit específico
git reflog  # Muestra un registro de todos los cambios de referencias (como cambios de ramas o reversiones)

8. Etiquetas (Tags)

git tag [etiqueta]  # Crea una etiqueta para el commit actual
git tag  # Lista todas las etiquetas en el repositorio
git push origin [etiqueta]  # Sube una etiqueta al repositorio remoto

9. Guardar Trabajos en Progreso

git stash  # Guarda temporalmente los cambios no confirmados
git stash apply  # Aplica los cambios guardados en el stash
git stash drop  # Elimina el stash más reciente

Ejemplos de Uso

1. Crear un Repositorio Nuevo

git init  # Crea un nuevo repositorio de Git en el directorio actual

2. Crear un Archivo y Hacer un Commit Inicial

echo "print('Hola Mundo')" > hola.py
git add hola.py
git commit -m "Commit inicial: Añadir hola.py"

3. Crear una Nueva Rama para Añadir una Funcionalidad

git branch nueva-funcionalidad
git checkout nueva-funcionalidad

4. Realizar Cambios en la Nueva Rama

echo "print('Funcionalidad añadida')" >> hola.py
git add hola.py
git commit -m "Añadir nueva funcionalidad a hola.py"

5. Fusionar la Nueva Rama con la Rama Principal

git checkout main
git merge nueva-funcionalidad

6. Eliminar la Rama

git branch -d nueva-funcionalidad

7. Publicar los Cambios en un Repositorio Remoto

git push origin main
